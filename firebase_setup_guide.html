If your evaluation tool is not saving to Firebase, you need to configure the Firestore security rules.

Step 1: Go to Firebase Console

Visit: https://console.firebase.google.com/

Select your project: ai-evaluation-82fc8

Step 2: Navigate to Firestore Database

In the left sidebar, click on "Firestore Database"

Click on the "Rules" tab at the top

Step 3: Update Security Rules

Replace the existing rules with these:

rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Allow read/write access to evaluations collection for authenticated users match /evaluations/{document} { allow read, write: if request.auth != null; } // Allow read access to test collection for connectivity testing match /test/{document} { allow read: if request.auth != null; } // Deny all other access match /{document=**} { allow read, write: if false; } } }

Step 4: Publish Rules

Click the "Publish" button to save the new rules

‚ö†Ô∏è Important Notes:

  • These rules allow any authenticated user to read and write to the evaluations collection
  • For production use, you might want more restrictive rules
  • The rules require users to be authenticated (which your app does with anonymous auth)

Step 5: Test the Connection

After updating the rules:

  1. Open your evaluation tool
  2. Open the browser console (F12)
  3. Look for these messages:
    • "Signed in anonymously to Firebase"
    • "Firebase read test successful"
    • "Firestore write successful!" (when saving)

Troubleshooting

If you still see errors in the console:

  • Permission denied: Rules are not published yet
  • Network error: Check your internet connection
  • Project not found: Verify the project ID in the Firebase config
